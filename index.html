B/S架构开发（二）—静态网页与动态网页

2022-08-01

静态网页

静态网页服务的实现首先需要客户机通过浏览器向服务器发出请求，然后服务器接受请求并根据请求从服务器端的网页中找到对应的页面，最后返回给客户机浏览器。这个过程中所发送的页面都是事先编辑好的，它并不能自动生成。

  • 静态网页不能自动更新，若要对静态页面进行更新，就要重新编写 HTML 源文件，然后再上传。因此静态网页的制作和维护工作量相当大。
  • 静态网页一经发布，无论浏览者浏览与否，它都是真实存在的一个文件，都对应一个 URL（文件在网上的地址）。

动态网页

动态网页是基本的 HTML 语法规范与 PHP、Java、Python 等程序语言、数据库等多种技术的融合，以期实现对网站内容和风格的高效、动态、交互式的管理。因此，可以理解为凡是结合了 HTML 以外的高级程序设计语言和数据库技术进行的网页编程技术生成的网页都是动态网页。

动态网页相对于静态网页来说，页面代码虽然没有变，但是显示的内容却是可以随着时间、环境或者数据库操作的结果而发生改变的。

动态网页可以是纯文字内容的，也可以包含各种动画的内容，这些只是网页具体内容的表现形式，无论网页是否具有动态效果，只要是采用了动态网站技术（如 PHP、JSP等）生成的网页都可以称为动态网页。

  • 动态网页在服务器端运行，客户机上看到的只是它的返回结果，不可能看到它的源文件。而静态网页则只能通过服务器把网页文件原封不动地传给客户机，本身不进行任何处理。
  • 动态网页只有经客户浏览时才会返回一个完整的网页，而其本身并不是一个独立存在于服务器的网页文件。
  • 与静态网页相比，动态网页更容易实现人机交互。与数据库相联系，能实现更为强大的功能。

如前所述，静态网页可以认为是最简单、最原始的Web服务形式，提供只读的网页文件供浏览器显示。而动态网页是从服务器端增强网页动态性，将HTML语言和PHP、JSP、ASP等脚本语言结合使用。那么能否从浏览器端（前端）提升网页的动态性呢？这就用到了JavaScript语言。

JavaScript是一门基于对象和事件驱动的脚本语言，通常用来提高网页与用户的交互性。JavaScript在用户的机器上而不是在服务器上执行任务，也被称为客户端语言。下一篇我们将重点解释引入这种客户端动态性如何实现动态网页。

B/S架构开发（一）—B/S架构简述

2022-07-11

C/S架构

与B/S架构相关的是C/S架构，即Client/Server（客户机/服务器）架构。C/S 架构是一种典型的两层架构，其客户端包含一个或多个在用户的电脑上运行的程序，进行界面显示与业务逻辑，而服务器端有两种，一种是数据库服务器端，客户端通过数据库连接访问服务器端的数据；另一种是Socket服务器端，服务器端的程序通过Socket与客户端的程序通信。

客户端要负责绝大多数的业务逻辑和UI展示，又称为胖客户端。它充分利用两端硬件，将任务分配到Client 和Server两端，降低了系统的通讯开销。C/S架构的软件需要针对不同的操作系统系统开发不同版本的软件。

C/S架构在技术上很成熟，它的主要特点是交互性强、网络通信量低、响应速度快、利于处理大量数据。升级维护和管理的难度大，不能实现快速部署安装和配置。

B/S架构

B/S架构，即Browser/Server（浏览器/服务器）架构，是目前应用系统的发展方向。B/S是伴随着Internet技术的兴起，对C/S架构的改进，为了区别于传统的C/S模式，特意称为B/S模式。B/S本质上也是C/S，只不过B/S架构中，使用浏览器作为软件的客户端，通过访问网页的形式来使用软件。

B/S架构相对于C/S架构有一些新的特点，熟悉C/S开发，并不一定能很自然的理解B/S开发。

首先，C/S架构主要采用两层结构，而B/S架构采用三层（3-tier）结构。 在这种结构下，我们通过浏览器进入工作界面，极少部分事务逻辑在浏览器端（Browser）实现，主要事务逻辑在服务器端（Server）实现。如下图所示。

一般我们称浏览器端为前端，Web服务器端为后端。信息流往往是：前端-后端-数据库-后端-前端。

其次，B/S架构中前后端代码都存放在后端， 浏览器只是提供代码解释运行的环境。也就是说不论前端代码（HTML、CSS、JavaScript）还是后端代码都是在后端存放，前端代码并不存放在前端。浏览器（前端）通过HTTP向后端请求相关的前端资源（网页资源）。这与C/S架构中客户端代码部署在客户端不同。前端、后端只是代表代码的运行环境。

B/S的主要特点是分布性强、维护方便、共享性强、总体拥有成本低。它是瘦客户端，对大量的数据输入以及报表的应答等都需要通过浏览器与服务器进行交互，通信开销大，而且对于实现复杂的应用构造有较大的困难。
